김규현, 빌궁팀 발표 시작하겠습니다.

--- 1p. ---

먼저 저희 프로젝트의 주제입니다.
저희 프로젝트의 주제는 커스텀 RDBMS 시스템 개발입니다.
주제의 내용으로는 SQL 쿼리 작성 및 실행, 데이터 저장 및 복구, 다중 사용자 실시간 시스템,
GUI 기반 테이블 생성 및 조작, 자연어 기반 AI 자동 쿼리 작성 기능 등을 지원하는
Rust로 개발된 풀스택 RDBMS 시스템 입니다.

--- 2p. ---

다음은 프로젝트 기능입니다.
크게 4가지로 분류할 수 있습니다.

첫번째로 RDBMS 시스템 기능입니다.
관계형 데이터베이스에서 SQL 쿼리문 작성을 통해 
데이터를 생성, 조회, 수정, 삭제 등을 할 수 있는 데이터베이스
엔진 기능입니다.

두번째로 네트워크 서버 기능입니다.
TCP 기반 서버를 생성하고, 인가된 다중 사용자가 데이터베이스에
접근하여 동시에 데이터를 조작할 수 있도록 하는 실시간 시스템 기능입니다.

세번째로 GUI 테이블 에디터 기능입니다.
GUI를 기반으로 SQL 쿼리문 없이 화면에서 테이블을 생성 및 조작하고,
관계를 설정할 수 있는 시각적 편집기 기능입니다.

마지막으로 AI 쿼리 작성 및 실행 기능입니다.
인공지능의 MCP 기술을 기반으로 비전문 사용자도 SQL 쿼리 작성 없이
간단한 명령어와 자연어를 통해 데이터를 조작할 수 있도록 하는 기능입니다.

--- 3p. ---

다음은 저희 프로젝트 평가 계획입니다.
큰 틀로는 4가지 핵심 기능을 안전하고 완전하게 사용 가능한 형태를 완성 기준으로 삼고 평가합니다.
중요도 순으로는 RDBMS 시스템, 서버, GUI, AI 순으로 평가 중요도를 나열하였습니다.

각 섹션별로 설명을 드리면 RDBMS 시스템 평가로는 실제 SQL 쿼리문을 작성하고 실행했을 때 정확한 결과가 나오는지
또한 데이터 변경 중 오류가 발생했을 때 데이터가 안전하게 복구되는지를 평가합니다.

다음은 서버 평가 항목입니다.
여러 사용자가 동시에 접속해서 같은 데이터베이스를 다룰 수 있는지
암호 등을 통해 인가받은 사용자만이 데이터베이스에 접근할 수 있는지를 평가합니다.

다음은 GUI 평가 항목입니다.
SQL 쿼리 없이도 시각적으로 테이블을 만들고 데이터를 조작할 수 있는지
테이블 간의 관계(외래키)를 시작으로 직접 설정하고 확인할 수 있는지를 평가합니다.

마지막으로 AI 평가 항목입니다.
자연어로 질의 쿼리를 작성했을 때 원하는 쿼리문으로 작성해주는지
AI로 생성한 자연어 기반 쿼리문이 올바른 로직을 수행하는지 평가합니다.

--- 4p. ---

다음은 진행 상황입니다.
먼저 진행중인 기능으로는
데이터베이스 엔진 고도화 및 최적화 작업,
네트워크 서버 인증 기능,
GUI 기반 테이블 편집기 설계,
UI/UX 개선 등을 진행하고 있습니다.

다음은 완료된 기능입니다.
사실 완료된 기능이 정말 많은데, 시간상 큰 틀로 설명드리겠습니다.
먼저 데이터베이스 엔진 기능 구현이 완료되었고,
데이터 저장, 복구 및 트랜잭션 기능이 완료되었습니다.
프론트 측면에서 UI/UX 클라이언트가 70% 완성된 상태이고,
TCP 포트 7878로 잡아서 네트워크 서버가 구현되었습니다.

다음은 가능성 확인 완료 기능입니다.
먼저 GUI 기반 테이블 에디터 같은 경우는 React Flow 기반으로 구현 가능성을 라이브러리 검토를 통해 확인하였습니다.
또한 AI 쿼리 기능에 대해서는 AI의 MCP API 연동 방식 및 SQL 변환 흐름 설계 가능성을 확인하였습니다.

--- 5p. ---

다음은 저희 프로젝트 문서의 산출물 입니다.
현재 프로젝트는 업무 분석표, 개발 목표 및 내용, 요구사항 정의, UI 명세, 기능적 모델링, 구조적 모델링,
클래스 메소드 알고리즘 명세, 주요 Use-Case에 대한 시퀀스 다이어그램 등을 작성한 프로젝트 통합 문서로 산출물을
관리하고 있습니다.

주로 문서에서 예정 섹션을 추가하여 작성하고 이에 따른 구현을 진행하고 있습니다.

다음은 저희 프로젝트의 중간 결과 동영상 시연입니다.

두번째로 GUI 기반 시각적 편집기 기능을 완성할 예정입니다.
